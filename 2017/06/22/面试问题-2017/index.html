面试问题-2017

发表于 2017-06-22
纪录一下2017年面试经历，目前算是2年工作经验了。但是从面试经历来看大部分还是被虐的😂 ，废话不多说了，看题吧。
First

本次面试面试官没有按照套路出牌，简历上写的东西一点都没问😂 。问题都是随想随问……

  1. 写出常用的字符串，时间，数组函数。
  2. 如何用php内置函数显示3个不同时区的时间。
  3. 知道哪些协议，并大体说一下。
  4. 对mongodb的基础操作：创建表，创建索引。
  5. 对mysql的基础操作：创建一个user表,包含姓名，性别，uid，密码。此处需要注意的就是主键，字段的类型，长度的设置。
  6. 对框架的理解，大体写一个简单的框架。
  7. socket编程。
  8. cgi与fastcgi。
  9. nginx关于php的location配置，关于静态文件的配置。

能记住的大体就是这些了。以上难点在于socket，框架，协议；其他我觉的还好。

Second

第二家面试的是小米，小米的办公环境有点厉害……感觉小米能成功还是有原因的，其内部整洁程度，对待外来人员的态度都是超好。看着还是很想进这样的公司的。
刚去的时候给了份面试题做，然后就是面试官根据简历问了一些东西。具体如下：
笔试题部分：

  1. 给定一个数组A和一个值sum，找出数组中所有和为sum的两个数。
  2. 816消消乐：给定一串数字B，然后对其进行消除。例如：81818166 其做完消除处理之后结果为81；818816166其做完消除处理之后结果为空。
  3. 给一张纪录学生成绩的mysql表，对其进行查询。比如：查出成绩总和最高的前5名。
  4. 给出了一段PHP程序。题目要求找出其中不合理的地方。

面试题部分：

  1. redis在项目中的使用场景。
  2. PDO是什么东西。
  3. 写出尽可能多的常用的数组函数并解释。
  4. error_reporting()，display_errors()。
  5. 安全方面

总结：

  • 笔试题1应该是大众算法，博客中有一篇简单的讲解了；
  • 笔试题2暂时没考虑出如何界；
  • 笔试题3很久没用mysql了只能放弃了；
  • 笔试题4这个还是很简单的；
  • PDO 提供了一个数据访问抽象层，这意味着，不管使用哪种数据库，都可以用相同的函数（方法）来查询和获取数据。
  • error_reporting()是用来设置应该报告何种错误的，display_errors 该选项设置是否将错误信息作为输出的一部分显示到屏幕，或者对用户隐藏而不显示。
