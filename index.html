尴尬的时间

什么情况下2-1=0？就是在你有两个选择的时候，其中有一个选择因为一些原因不能选，其实那你就没有选择了。

世事无常

  • 跳槽
  • 静静
跳槽

现在跳槽一方面工资可能会提高，但是我觉的很难到达20k的水准。原因有以下：

  • 工作经验，就目前工作经验算作2年，想要达到20k,还是有点难度，而且好一点的公司估计也不会出这么高的工资。
  • 个人能力，什么样的程序能到20k的水准，这个在我看来肯定是要自己负担过很大的项目，就目前公司的项目其实不够。

假如说跳槽，那下家公司会不会不加班？会不会有这么宽松的制度？还有交通问题。我觉得不想换工作有一部分原因是现在公司距离我住的地方近，每天不想打车，就可以走回家。这种生活过的时间长了就比较喜欢这种生活方式了。
还有一个主要的原因是，目前公司的技术负责人A是我比较想跟的人。不管是从写代码的能力，还是对问题的思考方式都值得我学习。如果跳槽，下家公司会不会遇到这样的人，这个很难说。

有一个好的负责人带是一件很重要的事，不仅可以快速提升你的技术能力,还会提高你的认知水平。A告诉我以我现在的能力想进BAT一类的一线互联网公司是很难的。学历是一方面，再就是没有说服人的能力。
这也是我不想跳槽的最大原因。过完年回公司的时候，本来就是打算在公司待到A辞职。

静静

A今天对我说了一下公司今年对于技术的规划可能不会很大，所以在资源上对技术的分配可能不是很多。所以，如果留在公司，那就意味着会有很多的空闲时间。

  • 如果利用好这段时间，那可能会带来本质的改变。A告诉我当时他在诺基亚时基本就和我现在的处境类似，然后A用这段时间的确做了很多该做的事，所以才会有了现在的能力。一个人能够到达什么样的高度，主要看这个人的自驱力，所谓自驱力是指一个人的自制力再加对自己的要求。如果我能在这段空闲的时间里，让自己能在技术和认知水平有一个明显的提高。那就不会由于在公司没事儿干，将已有的技术能力荒废掉。A说可能过几个月他也只会在这家公司兼职，出去继续做自己的事情。他想带着我，但是这种创业其实很渺茫。而且创业初期其实也没有多少的现金提供。我的担心主要是在现金方面，还有后续的发展空间能有多大。A也直说创业这种事就是千死一生的事儿。这个大家都明白,只不过在于你的胆识和眼光对不对。
  • 假如这段时间什么都没有做成，那等到A真的走了，那我就是损失了几个月的时间,晚了几个月找工作而已，而这几个月的时间过后找工作起薪我觉的应该会在15k。如果这样的话，最坏的发展就是到年中的时候工资还是现在这个水平，其实并没有什么大的损失。

结束语

A今天中午刚告诉我这件事的时候我有点发懵，很多事情没想过来。现在把事情分析一下，去留显而易见。

  • 很多事情不要去妄自下结论，首先要给事情定义一下严重程度。类似于今天的这个事儿应该可以说是对我个人很重要的事。因为比较喜欢走在路上的时候想事情，所以就直接下午从公司回家。边走边想，慢慢的你就会理清其中的主次。
  • 定义出事态的严重程度之后，再去分析事情最坏能够带来什么结果。这个结果能给你带来多大的影响，你能不能接受这样的结果。
  • 分析完最坏的结果，可以考虑一下出现概率最大的结果。这件事情出现概率最大的结果就是过了几个月A只在公司兼职，然后我可能会跟着A去工作，也可能会直接找下份工作。

突然发现这种分析问题的方式也是A在工作中最常用的方式…
其实，说到底还是没有足够的财富去任性的做事情。
所以只能已有的能力争取在短时间内积累大量财富了。

为什么要搭建个人博客

我觉的有以下两点理由就足以让我们自己动手去搭建了：

  • 个人博客有一种家的感觉。
  • 其他站点的博客丑到没朋友。太注重商业化。

如何搭建个人博客(以Mac为例)

github

  • 首先登录github，没有的可以注册一个（注：用户名不建议用数字，虽然后续可以绑定域名，但是我觉的绑定域名没有必要）。
  • 其次，新建一个仓库，该仓库的名字必须是username.github.io。该仓库的名字必须是username.github.io。该仓库的名字必须是username.github.io。重要的事情说三遍。此处的username要换成你自己的username！
  • 然后将仓库拉取到本地。
  • 最后登录github，然后选择username.github.io这个仓库，然后点击导航栏中的setting,会看到github已经自动为你生成了一个博客地址https://username.github.io/

hexo官方文档我觉的已经描述的非常详细了，在此就不赘述了。

docker 基本概念

镜像

docker镜像可以理解成一种文件系统，其除了提供容器运行时所需的程序，库，资源，配置等，还提供了为运行镜像时提供的配置参数。但要明白的一点是镜像不包含任何动态数据。可以参考代码中的类来理解。

容器

一开始我很不明白容器到底是个什么东西，看到有大神说如果把镜像比作类的话，那容器就是类的实体。
其实容器的本质是进程，但与直接在主机中运行的进程不同的是，容器进程运行于属于自己的命名空间。因此容器可以有自己的root文件系统、自己的进程空间，甚至自己的用户id空间。

仓库

仓库顾名思义就是存储的地方,类似于git的仓库，官方仓库里面有很多优质镜像

实践

实践的前提是你要了解docker的基本命令。
